# AI Web Crawler MCP Server

A Model Context Protocol (MCP) server that wraps the AI Web Crawler's Python Google Docs scraper and exposes it via FastMCP using the `@ai-universe/mcp-server-utils` helpers.

## Features

- Accepts Google Docs URLs and returns the combined Markdown generated by the existing Python crawler
- Supports crawling linked Google Docs up to a configurable depth
- Runs as an HTTP MCP server using the FastMCP HTTP stream transport
- Ready to publish to npm with an executable binary entry point

## Installation

```bash
npm install ai-web-crawler-mcp-server
```

This package requires Python 3 with the crawler dependencies from this repository (`requests`, `beautifulsoup4`, `markdownify`, `lxml`).

## Usage

### As a CLI

```bash
# From the repository root
npm install --prefix mcp-server
npm start --prefix mcp-server
```

By default the server listens on `http://0.0.0.0:8000/mcp`. Configure ports and origins using environment variables:

- `PORT` ŌĆō HTTP port for the Express proxy (default `8000`)
- `MCP_HOST` ŌĆō Host interface to bind (default `0.0.0.0`)
- `MCP_ALLOWED_ORIGINS` ŌĆō Comma separated list of allowed origins for CORS (default `*`)
- `MCP_PROXY_PATH` ŌĆō Path to mount the MCP endpoint (default `/mcp`)
- `MCP_HEALTH_PATH` ŌĆō Health check endpoint path (default `/healthz`)
- `CRAWLER_PYTHON_BIN` ŌĆō Python binary to execute (default `python3`)

### Tool definition

The server registers a single tool named `crawl_google_doc` with the following parameters:

| Parameter | Type | Description |
| --- | --- | --- |
| `url` | string | Google Docs URL to crawl (required) |
| `maxDepth` | integer | Maximum link depth to follow (default `1`, min `0`, max `5`) |
| `delay` | number | Delay in seconds between requests (default `1`, min `0`, max `10`) |

The tool returns two text messages:

1. The combined Markdown for all crawled documents (`mimeType: text/markdown`)
2. A plain-text crawl summary including discovered documents and crawler output

The HTTP proxy also exposes a JSON health check at `/healthz` (configurable via `MCP_HEALTH_PATH`), which is useful for deploy previews and uptime monitoring.

## Deploy previews

Use the root-level `Deploy preview (async)` GitHub Actions workflow together with the `github-actions-deploy` helper package to
trigger builds in VPC-SC restricted environments. The workflow posts rich progress updates back to the pull request and links to
the MCP health check once the preview environment is available.

### Embedding in other code

```js
import { startCrawlerServer } from 'ai-web-crawler-mcp-server';

await startCrawlerServer({
  port: 8080,
  host: '127.0.0.1'
});
```

This utility automatically configures FastMCP using `startFastMcpHttpProxy` from `@ai-universe/mcp-server-utils`.
